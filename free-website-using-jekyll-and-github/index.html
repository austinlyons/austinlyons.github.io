Free website using Jekyll and Github

Want to make a simple, decent-looking website so you can publish some
thoughts and code-snippets? Don't want to pay for hosting?
Try using Jekyll to build your website and host it with GitHub Pages. After the initial setup and configuration, publishing thoughts to a blog will be as simple as creating a text file on your laptop and saving it to the cloud. You don't even need to write HTML.

Here's how I did it.

Host your website with GitHub

If I create a repository on GitHub called austinlyons.github.io and use
it to store the contents of my website, GitHub will display my website
at http://austinlyons.github.io.

I'll walk you through creating a GitHub
repository that contains the source code for a basic website. The
website will be accessible (for free!) on the web at
your_github_username.github.io.

This part of the tutorial won't take long for people who are already familiar with git and have a GitHub account.
The rest can install git and signup for a GitHub account as we go - it
will take you a little longer to complete this section, but it's a great excuse
for you to learn about git and GitHub.

Create the source code repository at GitHub

Login to GitHub or create a free
account if you
don't have one yet. If you aren't familiar with GitHub, they are a company that lets you easily store
source code on their servers. If that doesn't mean much to you yet, no worries. You'll catch on quickly.

Next, create a repository named insertyourusernamehere.github.io. _Don't initialize the repository with a README, a .gitignore, or a license. Just leave everything blank
(to create a completely empty repository) and make sure it's public.

My GitHub username is
austinlyons, so I created austinlyons.github.io.

Create a local repository

Instead of writing the website from scratch, let's just take an existing
website and modify it to suit our needs. We'll use Jekyll
Bootstrap.

Open up your terminal (command prompt if you're running Windows). I use iTerm on Mac OS X. If you are on a Mac and don't know what I'm talking about, you can just use Terminal (type Terminal in spotlight). If you're using Linux, you already know what you're doing, so carry on. If you don't have Git installed already, now is the time to install Git.

If you've never heard of Git, I'm glad to be the one to introduce you. It's a simple distributed version control system - think Dropbox for programmers. You can learn more from Git's website. Note that Git and GitHub are two different things - Git is a program that you run on your computer, and GitHub is a company that specializes in hosting Git projects on their servers.

Let's copy the source code of the example website to our computer. I'll
use the dollar sign ($) to represent the terminal prompt (so you'll type
everything after the dollar sign).

We'll download the example website from the Jekyll
Bootstrap repository. Substitute your GitHub username wherever you see
austinlyons.

$ git clone https://github.com/plusjade/jekyll-bootstrap.git austinlyons.github.io

What we're doing here is downloading the source code of an existing
website (created with Jekyll) to our computer and storing it in a folder called austinlyons.github.io (your_username.github.io).
You should see something like:

Cloning into austinlyons.github.io...

remote: Compressing objects: 100% (951/951), done.
remote: Total 1868 (delta 921), reused 1720 (delta 804)
Receiving objects: 100% (1868/1868), 511.68 KiB | 307 KiB/s, done.
Resolving deltas: 100% (921/921), done.

Add the example code to your repository

Now that you've downloaded the example website to your computer, let's
copy it to GitHub. (Remember to substitute your username wherever you
see austinlyons). If you've never pushed code from your computer to a
GitHub repository using SSH, you'll need to follow the directions
provided by GitHub. If you
have trouble, tweet at me (referring to this tutorial) and I'll help you
out. 

$ cd austinlyons.github.io
$ git remote set-url origin git@github.com:austinlyons/austinlyons.github.io.git
$ git push origin master

We need to be inside the Git project before we can issue Git commands, so cd austinlyons.github.io moves us into the folder named austinlyons.github.io. (cd stands for change directory).

The second command git remote set-url origin tells git to forget about the
repository that we copied this code from and instead remember the URL
for our new repository's address. Then, when we run git push origin master,
git knows to send the code to the new repository that we just created.
(If we didn't do the second command above, git would try to push the code back to the original repository that we copied it from).

If you see something similiar to the following, you've successfully
copied the example website to your GitHub repository.

Your website is live!

It may take a few minutes, but you should be able to access your website
at http://your_username.github.io. In my case, GitHub shows this website
at http://austinlyons.github.io.

Edit your website with the help of Jekyll

We can make changes to your website locally (on your computer), and
then push them to GitHub.

Install Jekyll

Jekyll is a program that turns simple textfiles into a static website.
One cool feature is that Jekyll allows you to write your blog posts (etc) using
Markdown instead of HTML! (Markdown is super simple). Jekyll has some nice documentation at jekyllrb.com, I encourage you to check it out to learn more.

Jekyll is written with the programming language Ruby. To see if you have
Ruby installed, open up your terminal and type

$ which ruby

If Ruby is installed, you'll see a path to the particular directory that
holds the Ruby executable. You could also type

$ ruby --version

to see which version of Ruby you're running.
If you don't have Ruby, you'll want to install it.

Once we have Ruby installed, we'll install Jekyll. 

$ gem install jekyll

Run Jekyll

Let's fire up jekyll. First, we need to navigate to
austinlyons.github.io. We'll tell Jekyll to run a simple webserver which will
allow us to see our website in a browser while we make changes to it.

We'll use the --watch flag to tell Jekyll to watch for
changes to any of the source files and regenerate automagically

$ jekyll serve --watch

At this point you should be able to load your website at http://localhost:4000.

Edit your website

The first thing I did to this website was install a new theme. This
website uses the dinky theme. You can see the available themes
here.

To install the theme, let's open up another Terminal window, navigate to
austinlyons.github.io, and run

$ rake theme:install git="git://github.com/sodabrew/theme-dinky.git"

After it finishes installing, refresh your website at http://localhost:4000 and your website should have a new look.

Next, I edited _config.yml. In particular, I changed the title,
removed the author, edited the github and twitter usernames, and turned
off comments. I then moved all of the existing HTML files (like
archive.html) to the _drafts folder (just in case I want to reference
them in the future).

For more suggestions on how to customize your website, check out Jekyll
Bootstrap's
documentation
or Jekyll's documentation.

As you edit your website, you should be able to see your changes when
you refresh at http://localhost:4000. If your
changes aren't showing up, try killing the jekyll serve --watch
process by hitting Control+C in that terminal windown. Then run jekyll
serve --watch, open your browser, and try refreshing.

Pushing changes to GitHub

Now that you've made your local copy of your website look amazing, it's
time to update austinlyons.github.io. First we need to tell git about
the changes we've made, and then we'll send the updated files to GitHub.

From your root directory (austinlyons.github.io folder), type

$ git status

You'll see that git knows which files you've changed. We need to approve
this changes.

$ git add .

Now we commit the changes we've approved and write a little message to
remind us or others in the future what we just did.

$ git commit -m "Initial configuration of my awesome website"

Let's send the updated code to GitHub.

$ git push origin master

Now visit http://austinlyons.github.io.
As always, substitute your username for austinlyons here. Within a
couple of minutes you should see your updated website!

Bonus points: custom domain name

austinlyons.github.io is a fine domain name, but I already own the domain seeaustinhack.com.
I want my GitHub hosted website to be addressable via seeaustinhack.com.
Fortunately, GitHub makes it easy to set up a custom domain. Per these
instructions, I simply created a file named CNAME
which has the following line of text in it www.seeaustinhack.com.
I added this new file to my repository and sent it to GitHub. 

git add .
git commit -m 'Add CNAME record'
git push master origin

Then I logged in to GoDaddy (who I purchased the domain name from) and
updated my A record to 204.232.175.78. After waiting about 10 minutes,
I was able to visit www.seeaustinhack.com and see my GitHub hosted
website that I created with Jekyll.

The End

This tutorial was written with Markdown. The entire website was
generated with Jekyll and is hosted for free on GitHub. If you happened
to read this far and/or tried setting up a website with Jekyll and
GitHub, let me know!
